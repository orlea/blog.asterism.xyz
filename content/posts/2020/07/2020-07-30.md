## 環境

- Mastodon 3.2.0
- docker
- Elasticsearch 6.8.10
- sudachi 2.0.2

## 概要

EC2からLinodeに移行して、CPUクレジットとかEBSクレジットとかを気にせずに重い処理を流しっぱにしておけるようになりました。

で、以前一度試したけど使い勝手の悪さに対してコストが重すぎてやめたElasticsearchの再導入を思いつき。

一度バニラのmastodonでそのままES導入してみましたが、日本語そのまま入力すると単語ではなく1文字に分解されてクソでかため息を出しながらsudachi対応してくださった方のソースをあさりましたとさ。

## 本体改造

sudachiを使えるようにするため、また検索結果がソートされない動作を変更するためソースコードを書き換える必要があります。

私はdockerでmastodonを動かしているので、書き換えたソースをdocker volumeでマウントしました。

[github.com/orlea/mstdn.asterism.xyz/commit/...](https://github.com/orlea/mstdn.asterism.xyz/blob/9fd4d8a70042af1f9422d14d1d0b4a16ad971b3a/mastodon/docker-compose.yml)

(commitに変な差分混じってるけど許して)

### __app/chewy/statuses_index.rb__ は4行目以下`setting_index`を以下のように書き換え

[github.com/orlea/mstdn.asterism.xyz/commit/...](https://github.com/orlea/mstdn.asterism.xyz/blob/9fd4d8a70042af1f9422d14d1d0b4a16ad971b3a/mastodon/statuses_index.rb)

```
settings index: {
      refresh_interval: '15m',
      number_of_shards: '10',
      number_of_replicas: '0'
  }, analysis: {
    tokenizer: {
      sudachi_tokenizer: {
        type: 'sudachi_tokenizer',
        discard_punctuation: true,
      },
    },
    analyzer: {
      content: {
        filter: %w(
          lowercase
          cjk_width
          sudachi_part_of_speech
          sudachi_ja_stop
          sudachi_baseform
        ),
        tokenizer: 'sudachi_tokenizer',
        type: 'custom',
      },
    },
  }
```

### __search_service.rb__ は38行目`definition`を以下のように

[github.com/orlea/mstdn.asterism.xyz/commit/...](https://github.com/orlea/mstdn.asterism.xyz/blob/9fd4d8a70042af1f9422d14d1d0b4a16ad971b3a/mastodon/search_service.rb)

```
definition = parsed_query.apply(StatusesIndex.filter(term: { searchable_by: @account.id })).query(match: { 'text.stemmed': { query: @query, operator: 'and'}})

definition = definition.order(id: { order: 'desc' })
```
